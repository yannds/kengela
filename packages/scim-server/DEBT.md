# DEBT.md — @kengela/scim-server

> Le cœur est honnête sur ce qu'il ne fait pas encore. Ce qui manque ou est partiel figure ici.

| #   | Sujet                                              | Etat                                           | Probleme                                                                                                                                                                                                                                                                                                                                                     | Cible de migration                                                                                                      | Prio |
| --- | -------------------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- | ---- |
| 1   | Filtres SCIM                                       | `eq` sur `userName`/`externalId`/`displayName` | `userName eq "..."` (insensible à la casse) ET `externalId eq "..."` (Users), `displayName eq "..."` (Groups) sont interprétés via regex bornées (anti-ReDoS) — suffisant pour le validateur Entra. Les opérateurs riches (`co`, `sw`, `pr`, `and`/`or`, `[]` complexes) restent ignorés : un filtre non supporté renvoie une liste vide, jamais une erreur. | Parser de filtres SCIM complet (arbre AST borné) mappé vers le port.                                                    | P3   |
| 2   | Intégration HTTP réelle                            | ABSENT (par conception)                        | Le paquet est framework-agnostique : les handlers sont purs (`req parsée → réponse`). Aucun câblage transport (routes, `application/scim+json`, auth Bearer, dérivation du tenant).                                                                                                                                                                          | Adapter NestJS (ou Express) qui résout tenant + parse le corps + sérialise `ScimResponse`.                              | P1   |
| 3   | Extension enterprise + attributs riches            | NON PORTÉS dans le cœur                        | La sérialisation expose le core User (identité, e-mail, active, meta). L'extension enterprise (department/manager…), les adresses, téléphones, rôles — présents dans `KengelaScimUser` / `projectScimUser` (et DÉCRITS par `/Schemas`) — ne sont ni relus ni réémis par ce cœur.                                                                             | Étendre `ScimUserRow` + `toScimUser` (round-trip Entra/Okta) une fois la persistance d'attributs câblée dans l'adapter. | P2   |
| 4   | Tri (`sortBy`/`sortOrder`)                         | ABSENT                                         | La pagination est fournie ; l'ordre des résultats suit l'ordre d'insertion du store. RFC 7644 §3.4.2.3 (tri) non implémenté (déclaré `sort.supported=false` dans `/ServiceProviderConfig`).                                                                                                                                                                  | Ajouter des options de tri au port `ScimStore` + parsing des paramètres.                                                | P3   |
| 5   | Bulk (`/Bulk`) et ETag/versioning (`meta.version`) | ABSENT                                         | Opérations groupées (RFC 7644 §3.7) et concurrence optimiste (`If-Match`) non gérées (déclaré `bulk.supported=false` / `etag.supported=false`). `meta.version` n'est pas émis.                                                                                                                                                                               | À évaluer selon les IdP cibles ; non requis par Entra/Okta pour un provisioning basique.                                | P3   |
