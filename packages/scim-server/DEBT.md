# DEBT.md - @kengela/scim-server

> The core is honest about what it does not do yet. What is missing or partial is listed here.

| #   | Topic                                               | State                                         | Problem                                                                                                                                                                                                                                                                                                                                 | Migration target                                                                                                | Prio |
| --- | --------------------------------------------------- | --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- | ---- |
| 1   | SCIM filters                                        | `eq` on `userName`/`externalId`/`displayName` | `userName eq "..."` (case-insensitive) AND `externalId eq "..."` (Users), `displayName eq "..."` (Groups) are interpreted via bounded regex (anti-ReDoS) - enough for the Entra validator. The rich operators (`co`, `sw`, `pr`, `and`/`or`, complex `[]`) remain ignored: an unsupported filter returns an empty list, never an error. | Full SCIM filter parser (bounded AST) mapped to the port.                                                       | P3   |
| 2   | Real HTTP integration                               | ABSENT (by design)                            | The package is framework-agnostic: the handlers are pure (`parsed req -> response`). No transport wiring (routes, `application/scim+json`, Bearer auth, tenant derivation).                                                                                                                                                             | NestJS (or Express) adapter that resolves the tenant + parses the body + serializes `ScimResponse`.             | P1   |
| 3   | Enterprise extension + rich attributes              | NOT PORTED into the core                      | Serialization exposes the core User (identity, email, active, meta). The enterprise extension (department/manager...), addresses, phones, roles - present in `KengelaScimUser` / `projectScimUser` (and DESCRIBED by `/Schemas`) - are neither reread nor re-emitted by this core.                                                      | Extend `ScimUserRow` + `toScimUser` (Entra/Okta round-trip) once attribute persistence is wired in the adapter. | P2   |
| 4   | Sorting (`sortBy`/`sortOrder`)                      | ABSENT                                        | Pagination is provided; the result order follows the store's insertion order. RFC 7644 §3.4.2.3 (sorting) not implemented (declared `sort.supported=false` in `/ServiceProviderConfig`).                                                                                                                                                | Add sort options to the `ScimStore` port + parameter parsing.                                                   | P3   |
| 5   | Bulk (`/Bulk`) and ETag/versioning (`meta.version`) | ABSENT                                        | Bulk operations (RFC 7644 §3.7) and optimistic concurrency (`If-Match`) not handled (declared `bulk.supported=false` / `etag.supported=false`). `meta.version` is not emitted.                                                                                                                                                          | To be evaluated depending on the target IdPs; not required by Entra/Okta for basic provisioning.                | P3   |
