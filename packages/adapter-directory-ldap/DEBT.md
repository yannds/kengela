# DEBT.md - @kengela/adapter-directory-ldap

> The port is an airlock, not a hideout. What is wrapped and weak is listed here, with its target.

| #   | What is wrapped                 | State                                  | Problem                                                                                                                                                                                                                                                                                  | Migration target                                                                                                                                                    | Prio |
| --- | ------------------------------- | -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---- |
| 1   | `ldapts` client                 | wrapped via `LdapClientLike`           | We describe a NARROW surface (`bind`/`search`/`unbind`). A real `Client` from `ldapts` is supposed to be structurally compatible, proven here by the default factory (`new Client(...)` assigned to `LdapClientLike`), but NO test against a real directory exercises the full pipeline. | Add a CI job with an ephemeral LDAP directory (e.g. OpenLDAP/samba-ad container) + integration suite: LDAPS bind, real pagination, binary `objectGUID`, `memberOf`. | P1   |
| 2   | Manager reduced to CN           | wrapped (inherited from `iam-mapping`) | `profileFromLdap` reduces the manager's DN to its CN for lack of a second LDAP call to resolve the DN into an e-mail (known debt inherited from a real integration). The approval chain therefore stays approximate if the app expects an e-mail.                                        | Resolve the manager's DN via a second `search` (or a DN->mail cache) in a V2 of the source.                                                                         | P2   |
| 3   | TLS and timeouts                | wrapped                                | The default factory configures `rejectUnauthorized` and the timeouts, but TLS hardening (custom CA, pinning, client mTLS) is not exposed. A directory with an internal CA today requires going through an injected `clientFactory`.                                                      | Expose the advanced TLS options (CA, client cert) in `LdapConnectionConfig`, config-driven.                                                                         | P3   |
| 4   | Value normalization             | wrapped                                | Only `string`/`number`/`boolean`/`Buffer` are stringified; any other exotic shape returned by an alternative client is silently discarded (fail-closed). No versioned attribute schema.                                                                                                  | Log (at the caller level) the discarded attributes + validate the expected shape.                                                                                   | P3   |
| 5   | LDAP filter escaping (RFC 4515) | scope                                  | The adapter passes the `filter` VERBATIM to the client (proven by `security-ldap.test.ts`: no concatenation here, so no injection introduced). But no escaping helper is provided: an app that built a filter from unescaped user input would stay exposed to LDAP filter injection.     | Expose an `escapeLdapFilterValue()` (escapes `\ * ( ) NUL` into `\HH`) for callers that compose filters.                                                            | P3   |
