# DEBT.md — @kengela/adapter-directory-ldap

> Le port est un sas, pas une planque. Ce qui est enveloppe et faible figure ici, avec sa cible.

| #   | Ce qui est enveloppe                    | Etat                               | Probleme                                                                                                                                                                                                                                                                                                                          | Cible de migration                                                                                                                                                          | Prio |
| --- | --------------------------------------- | ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---- |
| 1   | Client `ldapts`                         | enveloppe via `LdapClientLike`     | On decrit une surface NARROW (`bind`/`search`/`unbind`). Un vrai `Client` de `ldapts` est cense etre structurellement compatible, prouve ici par la fabrique par defaut (`new Client(...)` affecte a `LdapClientLike`), mais AUCUN test contre un annuaire reel n'exerce le pipeline complet.                                     | Ajouter un job CI avec un annuaire LDAP ephemere (ex. conteneur OpenLDAP/samba-ad) + suite d'integration : bind LDAPS, pagination reelle, `objectGUID` binaire, `memberOf`. | P1   |
| 2   | Manager reduit au CN                    | enveloppe (herite d'`iam-mapping`) | `profileFromLdap` reduit le DN du manager a son CN faute de second appel LDAP pour resoudre le DN en e-mail (dette connue heritee d'une integration reelle). La chaine d'approbation reste donc approximative si l'app attend un e-mail.                                                                                          | Resoudre le DN du manager via un second `search` (ou un cache de DN->mail) dans une V2 de la source.                                                                        | P2   |
| 3   | TLS et timeouts                         | enveloppe                          | La fabrique par defaut configure `rejectUnauthorized` et les timeouts, mais le renforcement TLS (CA custom, pinning, mTLS client) n'est pas expose. Un annuaire a CA interne exige aujourd'hui de passer par un `clientFactory` injecte.                                                                                          | Exposer les options TLS avancees (CA, cert client) dans `LdapConnectionConfig`, config-driven.                                                                              | P3   |
| 4   | Normalisation des valeurs               | enveloppe                          | Seuls `string`/`number`/`boolean`/`Buffer` sont stringifies ; toute autre forme exotique renvoyee par un client alternatif est ecartee silencieusement (fail-closed). Pas de schema d'attributs versionne.                                                                                                                        | Journaliser (au niveau appelant) les attributs ecartes + valider la forme attendue.                                                                                         | P3   |
| 5   | Echappement des filtres LDAP (RFC 4515) | scope                              | L'adapter transmet le `filter` VERBATIM au client (prouve par `security-ldap.test.ts` : aucune concatenation ici, donc aucune injection introduite). Mais aucun helper d'echappement n'est fourni : une app qui construirait un filtre a partir d'entree utilisateur non echappee resterait exposee a l'injection de filtre LDAP. | Exposer un `escapeLdapFilterValue()` (echappe `\ * ( ) NUL` en `\HH`) pour les appelants qui composent des filtres.                                                         | P3   |
